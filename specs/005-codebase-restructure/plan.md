## Summary

Refactor the Bismuth codebase to enforce the 300-line constitution limit, split section-commented monolithic files into focused modules, modularize the Rust backend entry point, standardize folder organization, and ensure consistent Tailwind/token CSS integration across all UI components including canvas.

## Technical Context

**Language/Version**: TypeScript 5.x (frontend), Rust stable (Tauri 2.x backend)

**Primary Dependencies**: Svelte 4, Tauri 2, Tailwind CSS v4.3.0, tracing (Rust logging)

**Storage**: SQLite via `db.rs`, JSON files in `.bismuth/components/`

**Testing**: Vitest (frontend), cargo test (backend)

**Target Platform**: macOS, Windows, Linux (Tauri desktop)

**Project Type**: Desktop application (PKM editor)

**Performance Goals**: No regression ‚Äî refactoring only, no behavioral changes

**Constraints**: Zero breaking changes to public APIs; all existing imports must continue to resolve via barrel re-exports

**Scale/Scope**: ~30 frontend files + ~8 backend files exceed 300 lines and need splitting

## Execution Phases

### Phase 1: Critical Type System Split (Highest Impact)

Split `types/canvas.ts` (629 lines) into focused modules. This unblocks all other work since every canvas file imports from here.

**Files**: `src/lib/types/canvas/` (7 new files)
**Risk**: Import path breakage ‚Äî mitigated by barrel `index.ts`

### Phase 2: Backend Modularization

Split `main.rs` into `app/` module with state and handler groups. Split `db.rs` into sub-modules.

**Files**: `src-tauri/src/app/`, `src-tauri/src/db/`
**Risk**: Compilation errors ‚Äî mitigated by incremental extraction

### Phase 3: Frontend Store/Config Splits

Split `componentLibrary.ts`, `presets.ts`, and other stores exceeding 300 lines.

**Files**: Multiple store/config modules
**Risk**: Low ‚Äî pure code movement with re-exports

### Phase 4: Component Splitting (Large Svelte Files)

Extract logic from 770-line `CanvasWorkspaceInteractive.svelte`, 701-line `FileTree.svelte`, 643-line `GraphView.svelte`, and other large components.

**Files**: ~15 new helper modules + trimmed Svelte files
**Risk**: Svelte reactivity must be preserved ‚Äî logic extraction uses exported functions called from reactive blocks

### Phase 5: Tailwind/Token CSS Audit

Verify all canvas `var()` references map to `tokens.css`. Replace hardcoded colors. Confirm dark mode.

**Files**: Canvas component `<style>` blocks, `tokens.css`
**Risk**: Visual regression ‚Äî manual dark mode verification needed

### Phase 6: Integration Verification & Cleanup

Remove orphaned files, verify welcome screen, settings, logging. Run full validation from `quickstart.md`.

**Files**: Various cleanup across codebase
**Risk**: Low ‚Äî verification only
