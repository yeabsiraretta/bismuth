# Markdown Guide

Bismuth uses **Markdown** for all note content. Here's a quick reference.

## Basic Formatting

- `**bold**` → **bold**
- `*italic*` → *italic*
- `~~strikethrough~~` → ~~strikethrough~~
- `==highlight==` → ==highlight==

## Links

- **Wikilinks**: `[[11.01-welcome-to-bismuth|Welcome]]` → [[11.01-welcome-to-bismuth|Welcome]]
- **External**: `[Bismuth](https://bismuth.app)` → [Bismuth](https://bismuth.app)

## Lists

- Bullet lists with `-` or `*`
- Numbered lists with `1.`
- Task lists with `- [ ]` and `- [x]`

## Code

Use fenced code blocks with language identifiers for syntax highlighting.

## Callouts

> [!tip] Pro Tip
> Use [[11.03-zettelkasten-method|Zettelkasten linking]] to build connections between ideas.

## Frontmatter

Every note can have YAML frontmatter for metadata:
```yaml
---
title: My Note
tags: [topic-a, topic-b]
date: 2026-01-01
---
```

See [[30-39-reference/31.02-frontmatter-reference|Frontmatter Reference]] for all supported fields.

## Flashcards

What syntax creates a wikilink in Bismuth? :: `[[note-name]]` or `[[path/to/note|Display Text]]`

Bismuth callouts use the syntax ==> [!type] Title== inside a blockquote.

How do you create a task in Markdown? :: Use `- [ ]` for unchecked and `- [x]` for checked items

Frontmatter ::: YAML metadata block at the top of a note enclosed in `---` delimiters

### What are the four main Bismuth-specific Markdown extensions? #card #markdown #syntax
Wikilinks (`[[note]]`), callouts (`> [!type]`), task checkboxes (`- [ ]`/`- [x]`), and frontmatter YAML blocks (`---`).
