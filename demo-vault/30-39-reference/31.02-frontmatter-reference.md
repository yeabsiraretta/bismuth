# Frontmatter Reference

Every Bismuth note can have YAML frontmatter at the top of the file, enclosed in `---`.

## Standard Fields

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Display name of the note |
| `date` | date | Creation or publish date |
| `tags` | list | Categorization tags |
| `id` | string | Unique identifier (Zettelkasten ID) |
| `type` | string | Note classification |
| `aliases` | list | Alternative names for [[wikilink]] resolution |

## Custom Properties

You can add any key-value pair. Bismuth indexes all properties for search and filtering.

```yaml
---
title: My Research Note
date: 2026-01-15
tags: [research, biology]
status: draft
priority: high
author: Jane Doe
---
```

## Property Search

Use the [[20-29-projects/21.02-command-palette|Command Palette]] to search by property:
- `property:status=draft` — Find all drafts
- `property:priority=high` — Find high-priority items

## Flashcards

What frontmatter field provides alternative names for wikilink resolution? :: `aliases` — a list of alternative names the note can be referenced by

How do you search by property in Bismuth? :: Use the Command Palette with `property:key=value` syntax (e.g., `property:status=draft`)

The ==aliases== frontmatter field allows a note to be found by alternative names in wikilink resolution.

Frontmatter `type` field ::: Classifies the note (e.g., Resource, Concept, Project, Person) for entity-based organization

### What are the most important frontmatter fields for flashcard grouping? #card #reference #frontmatter
`type` (Resource, Concept, Project, Person, Event, Task) for entity-based grouping, and `tags` for topic-based grouping — both enable filtering flashcards across the vault.
