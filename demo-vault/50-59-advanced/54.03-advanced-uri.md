# Advanced URI

Control Bismuth programmatically using the `bismuth://` URI scheme, similar to Obsidian Advanced URI.

## URI Format

```text
bismuth://action?param1=value1&param2=value2
```

## Supported Actions

### Open a Note

```text
bismuth://open?vault=MyVault&filepath=notes/todo.md
bismuth://open?filepath=note.md&heading=Goals
bismuth://open?filepath=note.md&block=abc123
bismuth://open?filepath=note.md&line=42
```

### Edit a Note

```text
bismuth://edit?filepath=note.md&data=Hello%20World&mode=append
bismuth://edit?filepath=note.md&clipboard=true&mode=prepend
```

Write modes: `append`, `prepend`, `overwrite`.

### Create a Note

```text
bismuth://create?filepath=new-note.md&data=Initial%20content
```

### Open Daily Note

```text
bismuth://daily
bismuth://daily?data=Meeting%20notes&mode=append
```

### Execute a Command

```text
bismuth://command?commandid=publish:site
```

### Search

```text
bismuth://search?query=project%20plan
bismuth://searchreplace?filepath=note.md&search=old&replace=new
```

### Frontmatter

```text
bismuth://frontmatter?filepath=note.md&frontmatterkey=status&data=active
```

### Canvas Navigation

```text
bismuth://canvas?canvasid=abc&x=100&y=200&zoom=1.5
```

### Workspace & Bookmarks

```text
bismuth://workspace?workspace=writing
bismuth://bookmark?bookmark=Research
```

### Annotate a PDF

```text
bismuth://annotate?filepath=papers/arxiv.pdf
```

## Deep Link Integration

URIs work from:

- **OS deep links** — Click `bismuth://` links from other apps, browsers, or scripts
- **In-app triggers** — Programmatically dispatch URIs within Bismuth
- **Automation** — Use in shell scripts, Alfred workflows, or Raycast commands

## Building URIs

URIs are built with `buildAdvancedUri(action, params)` and parsed with `parseAdvancedUri(uriString)`. Both functions handle URL encoding automatically.

> [!tip] Testing URIs
> Use the command palette to test: run any command, then construct a URI to trigger it externally.
