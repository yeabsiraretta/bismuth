# Bismuth Architecture

Bismuth is built on a **Tauri + Svelte + Rust** stack for native performance with web-based UI.

## Layers

1. **Frontend** — Svelte 5, TypeScript, Canvas API
2. **IPC Bridge** — Tauri commands (type-safe RPC)
3. **Backend** — Rust services, SQLite database
4. **Filesystem** — Vault stored as plain Markdown files

> [!warning] Architecture Constraint
> All IPC calls must go through service layers — stores must never call `invoke()` directly.

## Key Services

- **Vault Service** — File CRUD, scanning, watching
- **Search Service** — Full-text indexing via Tantivy
- **Graph Service** — Knowledge graph from [[wikilink|wikilinks]] and backlinks
- **Template Service** — Note creation from [[30-39-reference/31.04-templates|templates]]
- **Canvas Service** — Infinite canvas with component library
- **Task Service** — Parses tasks from markdown with Obsidian Tasks syntax
- **Entity Service** — Typed notes (Project, Concept, Person, etc.)
- **Longform Service** — Multi-scene manuscript projects
- **Publishing Service** — Static site generation from vault content
