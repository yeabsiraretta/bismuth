# AWS Well-Architected Framework

Six pillars for building reliable, secure, efficient, cost-effective, and sustainable systems on [[50-59-aws-saa/51.00-aws-saa-overview|AWS]].

## 1. Operational Excellence

Run and monitor systems to deliver business value. Improve processes continuously.

- Infrastructure as Code ([[50-59-aws-saa/51.20-aws-extra-services#CloudFormation|CloudFormation]])
- Annotate documentation
- Make frequent, small, reversible changes
- Anticipate failure

Services: CloudFormation, [[50-59-aws-saa/51.19-aws-monitoring|CloudWatch]], AWS Config, CloudTrail, X-Ray.

## 2. Security

Protect information, systems, and assets. Implement strong [[50-59-aws-saa/51.01-aws-iam|identity management]].

- [[50-59-aws-saa/51.01-aws-iam|IAM]]: least privilege, MFA, roles
- Enable traceability ([[50-59-aws-saa/51.19-aws-monitoring#CloudTrail|CloudTrail]])
- Apply security at all layers ([[50-59-aws-saa/51.16-aws-vpc|VPC]], [[50-59-aws-saa/51.02-aws-ec2-fundamentals#Security Groups|SGs]], NACLs)
- Automate security best practices
- Protect data in transit and at rest ([[50-59-aws-saa/51.15-aws-security#KMS|KMS]])
- Prepare for security events

## 3. Reliability

System's ability to recover from disruptions and dynamically meet demand.

- Test recovery procedures
- Automatically recover from failure ([[50-59-aws-saa/51.06-aws-asg|ASG]], [[50-59-aws-saa/51.05-aws-elb|ELB]], multi-AZ)
- Scale horizontally for aggregate system availability
- Stop guessing capacity
- Manage change in automation

Services: [[50-59-aws-saa/51.10-aws-route53|Route 53]], [[50-59-aws-saa/51.06-aws-asg|ASG]], [[50-59-aws-saa/51.05-aws-elb|ELB]], [[50-59-aws-saa/51.09-aws-databases|RDS]] Multi-AZ, [[50-59-aws-saa/51.08-aws-s3|S3]] replication.

## 4. Performance Efficiency

Use computing resources efficiently and maintain as technology evolves.

- Democratize advanced technologies (managed services)
- Go global in minutes ([[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]], multi-region)
- Use serverless architectures ([[50-59-aws-saa/51.13-aws-serverless|Lambda]])
- Experiment more often

Services: [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] (right-sizing), [[50-59-aws-saa/51.13-aws-serverless|Lambda]], [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]], [[50-59-aws-saa/51.09-aws-databases|ElastiCache]].

## 5. Cost Optimization

Deliver business value at the lowest price point.

- Adopt a consumption model (pay for what you use)
- Measure overall efficiency
- Stop spending on data center operations
- Analyze and attribute expenditure
- Use managed and application-level services

Services: [[50-59-aws-saa/51.02-aws-ec2-pricing|Reserved Instances, Spot Instances]], [[50-59-aws-saa/51.06-aws-asg|ASG]], [[50-59-aws-saa/51.08-aws-s3|S3 Lifecycle]], [[50-59-aws-saa/51.13-aws-serverless|Lambda]].

## 6. Sustainability

Minimize environmental impact.

- Understand your impact
- Establish sustainability goals
- Maximize utilization
- Use managed services (shared infrastructure)
- Reduce downstream impact

## Well-Architected Tool

Free tool in AWS Console. Review workloads against the 6 pillars. Get advice and improvement plans.

## Flashcards

What are the six pillars of the Well-Architected Framework? :: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, and Sustainability

The ==Reliability== pillar focuses on recovering from failures and meeting demand, while ==Operational Excellence== focuses on running and monitoring systems to deliver business value.

Well-Architected Tool ::: Free AWS Console tool that reviews workloads against the 6 pillars, providing advice and improvement plans

The Sustainability pillar aims to minimize the ==environmental impact== of running cloud workloads by maximizing utilization and reducing downstream resource needs.

### What does each Well-Architected pillar focus on? #card #aws #well-architected

Operational Excellence: automate operations, respond to events. Security: protect data, systems, and assets. Reliability: recover from failures, scale to meet demand. Performance Efficiency: use resources efficiently. Cost Optimization: avoid unnecessary costs. Sustainability: minimize environmental impact.

Operational Excellence design principles :: Perform operations as code (IaC), make frequent small reversible changes, refine procedures frequently, anticipate failure, learn from operational failures

Security design principles :: Implement strong identity foundation, enable traceability, apply security at all layers, automate security best practices, protect data in transit and at rest, keep people away from data, prepare for security events

Reliability design principles :: Automatically recover from failure, test recovery procedures, scale horizontally, stop guessing capacity, manage change in automation

Performance Efficiency design principles :: Democratize advanced technologies, go global in minutes, use serverless architectures, experiment more often, consider mechanical sympathy

Cost Optimization design principles :: Implement cloud financial management, adopt a consumption model, measure overall efficiency, stop spending money on data center operations, analyze and attribute expenditure

### What AWS service helps with each Well-Architected pillar? #card #aws #well-architected

Operational Excellence: CloudFormation, Config, CloudWatch. Security: IAM, KMS, Shield, WAF, GuardDuty. Reliability: ASG, Multi-AZ, Route 53. Performance Efficiency: Lambda, CloudFront, ElastiCache. Cost Optimization: Reserved Instances, S3 Intelligent-Tiering, Trusted Advisor. Sustainability: EC2 Auto Scaling, Graviton instances.

AWS Trusted Advisor :: Service that provides real-time guidance following AWS best practices across 5 categories — cost optimization, performance, security, fault tolerance, and service limits
