# AWS VPC & Networking

Virtual Private Cloud — isolated network within AWS.

## CIDR - IPv4

Classless Inter-Domain Routing. Two components:

- **Base IP** — e.g., `10.0.0.0`, `192.168.0.0`
- **Subnet Mask** — `/n` means n bits are fixed. Addresses = 2^(32-n).

### Private IP Ranges (RFC 1918)

- `10.0.0.0/8` — big networks
- `172.16.0.0/12` — AWS default VPC range
- `192.168.0.0/16` — home networks

## VPC Basics

- Max **5 VPCs per region** (soft limit)
- Max 5 CIDRs per VPC (min /28 = 16 IPs, max /16 = 65,536 IPs)
- Only private IPv4 ranges allowed
- VPC CIDR must NOT overlap with other networks

### Default VPC

Allows immediate [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] deployment. All subnets have internet route. Each EC2 gets public + private IP.

## Subnets

AWS reserves **5 IPs** per subnet (first 4 + last 1): network address, VPC router, DNS, future use, broadcast.

**Exam Tip**: Need 29 IPs? Can't use /27 (32-5=27 < 29). Use /26 (64-5=59).

## Internet Gateway (IGW)

Allows [[50-59-aws-saa/51.16-aws-vpc|VPC]] resources to connect to internet. One VPC = one IGW. Must also edit route tables.

## Bastion Hosts

Public subnet EC2 that allows SSH into private subnet instances. Tighten SG — only port 22 from your IP.

## NAT (Network Address Translation)

Allow private subnets to access internet (one-way: no inbound connections).

### NAT Instance (Legacy)

- Launch in public subnet with Elastic IP
- Must manage SGs; not highly available
- Disable source/destination check

### NAT Gateway (Preferred)

- AWS-managed, highly available within AZ, auto-scaling
- Elastic IP required. No SGs.
- 5-45 Gbps bandwidth
- Can't span AZs — create one per AZ for HA
- Can't be used by EC2 in same subnet

### Egress-Only Internet Gateway

Same as NAT but for **IPv6**. Refuses inbound connections without outbound history.

## DNS Attributes

- `enableDnsHostnames` — assigns public DNS to instances with public IPs (default false)
- `enableDnsSupport` — queries Amazon DNS / [[50-59-aws-saa/51.10-aws-route53|Route 53]] (default true)

## Network ACLs (NACLs)

Firewall at **subnet level**. One NACL per subnet.

- Rules numbered 1-32766, **lower number = higher priority**
- First match wins; last rule (*) denies all
- AWS recommends increments of 100
- **Default NACL** allows all inbound/outbound — do NOT modify, create custom
- **Custom NACLs** deny all by default
- NACLs act **before** [[50-59-aws-saa/51.02-aws-ec2-fundamentals#Security Groups|Security Groups]]
- Can block **individual IPs** (unlike SGs)
- Ephemeral ports (1024-65535) — must allow both inbound and outbound

## VPC Peering

Private connection between two VPCs. No transitive peering (each pair needs own link).

- Can peer across regions/accounts
- **Cannot** peer VPCs with overlapping CIDRs
- Route tables must be updated in all peered VPCs
- Does NOT support edge-to-edge routing

## VPC Endpoints (PrivateLink)

Connect to AWS services via **private network** (no IGW, NAT, VPN, DX needed).

### Interface Endpoints

- ENI (private IP) as entry point, must attach SG
- Supports most AWS services

### Gateway Endpoints

- Target in route table (like NAT Gateway)
- **Only [[50-59-aws-saa/51.08-aws-s3|S3]] and DynamoDB**
- Preferred for S3/DynamoDB (unless on-prem or cross-VPC access needed)

## VPC Flow Logs

Capture IP traffic info. Three levels: VPC, Subnet, ENI. Store in [[50-59-aws-saa/51.19-aws-monitoring|CloudWatch]] or [[50-59-aws-saa/51.08-aws-s3|S3]].

- `srcaddr`/`dstaddr` — identify problematic IPs
- `srcport`/`dstport` — identify problematic ports
- Query with [[50-59-aws-saa/51.18-aws-data-analytics#Athena|Athena]]

## Flashcards

What is the difference between NACLs and Security Groups? :: NACLs are stateless (must allow both inbound and outbound), operate at subnet level with allow/deny rules; Security Groups are stateful, operate at instance level with allow-only rules

A VPC spans ==one Region== and subnets span ==one AZ==.

VPC Endpoints ::: Private connections from VPC to AWS services without traversing the internet — Gateway Endpoints (S3, DynamoDB) and Interface Endpoints (all other services via PrivateLink)

The default NACL ==allows== all inbound and outbound traffic; a custom NACL ==denies== everything by default.

### Security Groups vs NACLs — key differences? #card #aws #vpc

Security Groups: stateful (return traffic auto-allowed), instance-level, allow rules only. NACLs: stateless (must explicitly allow return traffic), subnet-level, allow AND deny rules, processed in order.

Max ==5 VPCs== per region (soft limit). Max ==5 CIDRs== per VPC with range /28 to /16.

AWS reserves ==5 IP addresses== per subnet — the first 4 and the last 1.

If you need 29 EC2 IPs in a subnet, /27 gives only ==27 usable== (32-5). You must use ==/26== which gives 59 usable (64-5).

Internet Gateway (IGW) :: Allows VPC resources to connect to the internet — one IGW per VPC, must edit route tables to enable connectivity

Bastion Host :: A public-subnet EC2 used to SSH into private-subnet instances — tighten SG to only allow port 22 from your IP

NAT Gateway :: AWS-managed service allowing private subnets to access the internet (outbound only) — requires Elastic IP, 5-45 Gbps bandwidth, create one per AZ for HA

NAT Gateway can't be used by instances in the ==same subnet== it's deployed in.

For IPv6 outbound-only access from private subnets, use an ==Egress-Only Internet Gateway== instead of NAT.

NACL rules are evaluated in order from ==lowest number to highest== — first match wins, and the last rule (*) denies everything.

NACLs must explicitly allow ==ephemeral ports== (1024-65535) for return traffic since they are stateless.

NACLs can block ==specific IP addresses==, which Security Groups cannot do.

VPC Peering is ==NOT transitive== — each VPC pair needs its own peering connection.

VPC Peering ==cannot== work between VPCs with overlapping CIDR ranges.

Gateway Endpoints only support ==S3== and ==DynamoDB== — all other services use Interface Endpoints (PrivateLink).

### What information can VPC Flow Logs capture? #card #aws #vpc

Source/destination IP addresses (srcaddr/dstaddr) and ports (srcport/dstport) for all network traffic. Can be enabled at VPC, Subnet, or ENI level. Stored in CloudWatch Logs or S3, queryable with Athena.

`enableDnsHostnames` assigns public DNS to instances with public IPs (default ==false==). `enableDnsSupport` enables Amazon DNS queries (default ==true==).

The three private IP ranges (RFC 1918) :: `10.0.0.0/8` (big networks), `172.16.0.0/12` (AWS default VPC), `192.168.0.0/16` (home networks)
