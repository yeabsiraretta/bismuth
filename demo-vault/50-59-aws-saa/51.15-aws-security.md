# AWS Security & Encryption

Security services and encryption across [[50-59-aws-saa/51.00-aws-saa-overview|AWS]].

## Encryption 101

- **In-flight (SSL/TLS)** — encrypted before sending, decrypted after receiving. Protects from man-in-the-middle.
- **At rest** — encrypted on server, decrypted before sending. Keys managed somewhere server can access.
- **Client-side** — encrypted by client, never decrypted by server. Could use envelope encryption.

## KMS - Key Management Service

AWS-managed encryption keys. Integrated with [[50-59-aws-saa/51.01-aws-iam|IAM]]. Audit via [[50-59-aws-saa/51.19-aws-monitoring#CloudTrail|CloudTrail]]. **Never store secrets in plaintext.**

### Key Types

- **Symmetric (AES-256)** — single key, used by most AWS services. Can't access unencrypted key (must call KMS API).
- **Asymmetric (RSA/ECC)** — public/private pair. Public downloadable. Use for encryption outside AWS.

### KMS Key Types

| Type | Cost |
|------|------|
| AWS Owned (SSE-S3, SSE-SQS, SSE-DDB) | Free |
| AWS Managed (`aws/service-name`) | Free |
| Customer Managed (KMS-created) | $1/month |
| Customer Managed (imported, symmetric only) | $1/month |
| API calls | $0.03/10,000 |

### Key Rotation

- AWS Managed: auto every **1 year**
- Customer Managed: auto every 1 year (must enable)
- Imported: manual only (use alias)

### KMS Key Policies

Control access to keys (like [[50-59-aws-saa/51.08-aws-s3|S3]] bucket policies). Cannot control access without them.
- **Default** — allows everyone in your account
- **Custom** — define specific users/roles (great for cross-account via [[50-59-aws-saa/51.04-aws-ec2-storage|EBS]] snapshot sharing)

### Cross-Region

Keys are **region-scoped**. To move encrypted data: KMS ReEncrypt with new region's key. Multi-region keys exist but managed independently.

### Encrypted AMI Sharing

Source account → Target account: modify image attribute (Launch Permission), share KMS key, grant DescribeKey/ReEncrypted/CreateGrant/Decrypt to target. Target can re-encrypt with own key.

## SSM Parameter Store

Secure storage for config and secrets. Optional KMS encryption. Serverless, versioned.

| Feature | Standard | Advanced |
|---------|----------|----------|
| Parameters per region | 10,000 | 100,000 |
| Max size | 4 KB | 8 KB |
| Parameter policies | No | Yes |
| Cost | Free | $0.05/param/month |

- Store in hierarchy (folder-like paths)
- Access Secrets Manager via path: `/aws/reference/secretsmanager/secret_ID`
- **Parameter policies** — assign TTL (password expiration), notification if unchanged

## Secrets Manager

- **Force rotation** every X days (uses [[50-59-aws-saa/51.13-aws-serverless|Lambda]] for rotation)
- Integration with [[50-59-aws-saa/51.09-aws-databases|RDS]] — auto-create DB credentials
- Encrypted with KMS
- **Multi-region secrets** — read replicas synced with primary, can promote replica

## AWS Certificate Manager (ACM)

Provision, manage, deploy TLS certificates. Free for public certs. Auto-renewal.

Integrates with: [[50-59-aws-saa/51.05-aws-elb|ELB]] (CLB, ALB, NLB), [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]], API Gateway. **Cannot use with [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]]**.

- Validation: DNS (CNAME, better for automation) or email
- Imported certs don't get auto-renewed (daily notifications 45 days before expiry)
- Edge-optimized API Gateway: cert must be in us-east-1; Regional: cert in same region as API

## WAF - Web Application Firewall

Protect from common web exploits (Layer 7 - HTTP).

Deploy on: [[50-59-aws-saa/51.05-aws-elb|ALB]], API Gateway, [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]], AppSync, Cognito.

- Web ACL rules: IP filtering (up to 10K IPs), HTTP header/body/URI inspection, SQL injection, XSS, geo-match, rate-based (DDoS)
- Rule Groups — reusable sets
- Regional except on CloudFront

## Shield

DDoS protection.

- **Standard** — free, automatic, protects from SYN/UDP floods, reflection attacks (L3/L4)
- **Advanced** — $3,000/month/org. Protects [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]], [[50-59-aws-saa/51.05-aws-elb|ELB]], [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]], Global Accelerator, [[50-59-aws-saa/51.10-aws-route53|Route 53]]

## Firewall Manager

Manage WAF, Shield, SG rules across all accounts in [[50-59-aws-saa/51.01-aws-iam-advanced#AWS Organizations|AWS Organizations]]. Policies applied to new resources automatically.

## GuardDuty

Intelligent threat discovery. Analyzes [[50-59-aws-saa/51.19-aws-monitoring#CloudTrail|CloudTrail]], VPC Flow Logs, DNS logs. Good for crypto-mining detection. Notifications via [[50-59-aws-saa/51.19-aws-monitoring#EventBridge|EventBridge]].

## Inspector

Automated security assessments — continuous CVE scanning + network reachability.
- [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]]: OS vulnerabilities, network accessibility
- ECR containers: scanned on push
- [[50-59-aws-saa/51.13-aws-serverless|Lambda]]: code + dependency vulnerabilities

## Macie

Data security service — alerts on sensitive data (PII) in [[50-59-aws-saa/51.08-aws-s3|S3]].

## Flashcards

What is the difference between SSM Parameter Store and Secrets Manager? :: Secrets Manager auto-rotates secrets (RDS, Redshift, DocumentDB), costs more; Parameter Store is free tier for standard, manual rotation via Lambda

KMS ==Symmetric== keys never leave KMS unencrypted (API only); KMS ==Asymmetric== keys let you download the public key for external encryption.

AWS WAF ::: Web Application Firewall that filters HTTP requests at Layer 7 — deployed on ALB, API Gateway, CloudFront, AppSync — blocks by IP, geo, rate, SQL injection, XSS

What does Amazon Macie do? :: Uses machine learning to discover and protect sensitive data (PII) in S3 buckets

### SSE-S3 vs SSE-KMS vs SSE-C — when to use each? #card #aws #security
SSE-S3: simplest, AWS-managed keys, no audit trail. SSE-KMS: customer-managed keys with CloudTrail audit + rotation control. SSE-C: customer provides encryption key with every request — AWS never stores the key.

AWS Owned KMS keys are ==free==; AWS Managed keys are ==free==; Customer Managed keys cost ==$1/month==.

KMS keys are ==region-scoped==. To move encrypted data cross-region, you must ReEncrypt with a key in the new region.

AWS Managed KMS keys rotate automatically every ==1 year==.

Customer Managed KMS keys require enabling rotation and then rotate every ==1 year==. Imported keys require ==manual== rotation using aliases.

### How do you share an encrypted AMI cross-account? #card #aws #security
1) Modify Launch Permission on the AMI for target account. 2) Share the KMS key used for encryption. 3) Grant target account DescribeKey, ReEncrypted, CreateGrant, and Decrypt permissions on the KMS key. Target can re-encrypt with own key.

SSM Parameter Store stores parameters in a ==hierarchy== (folder-like paths) and can reference Secrets Manager via `/aws/reference/secretsmanager/secret_ID`.

SSM Parameter Store Standard tier is ==free== (10,000 params, 4KB max). Advanced tier costs ==$0.05/param/month== (100,000 params, 8KB max).

Secrets Manager forces rotation every X days using ==Lambda== for rotation logic.

Secrets Manager Multi-Region Secrets :: Read replicas synced with a primary secret across regions — can promote a replica to standalone for disaster recovery

ACM (AWS Certificate Manager) :: Provision, manage, and deploy free public TLS certificates with auto-renewal — integrates with ELB, CloudFront, and API Gateway but NOT with EC2

ACM DNS validation uses a ==CNAME== record and is preferred for automation; email validation requires manual approval.

For edge-optimized API Gateway, ACM certs must be in ==us-east-1==. For regional API Gateway, certs must be in the ==same region==.

Shield Standard is ==free== and automatic; Shield Advanced costs ==$3,000/month/org== with DDoS response team and cost protection.

Firewall Manager :: Manages WAF, Shield, and SG rules across all accounts in AWS Organizations — policies automatically apply to new resources

GuardDuty :: Intelligent threat discovery that analyzes CloudTrail, VPC Flow Logs, and DNS logs — good for detecting cryptocurrency mining attacks

Inspector :: Automated security assessments for EC2 (OS vulnerabilities), ECR containers (scanned on push), and Lambda functions (code + dependency vulnerabilities)

### WAF vs Shield vs Firewall Manager — when to use each? #card #aws #security
WAF: filter HTTP requests at Layer 7 with web ACL rules (IP, geo, rate, SQL injection). Shield: DDoS protection at Layer 3/4 (Standard free, Advanced $3K/mo). Firewall Manager: centrally manage WAF/Shield/SG rules across all accounts in AWS Organizations.
