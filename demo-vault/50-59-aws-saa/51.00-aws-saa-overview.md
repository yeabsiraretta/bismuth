# AWS Solutions Architect Associate - Overview

Study notes for the **AWS Certified Solutions Architect - Associate** exam, following the Udemy course by Stephane Maarek.

## Exam Format

- 50 graded questions + 15 ungraded
- 130 minutes
- No penalty for guessing
- Aim for 70-75% on practice exams before attempting
- Very few trick questions; overcomplicated answers are usually wrong

## Global Infrastructure

### Regions

A geographical area consisting of 2+ [[50-59-aws-saa/51.00-aws-saa-overview#Availability Zones|Availability Zones]].

How to choose a region:
- **Compliance** — data governance and legal requirements
- **Proximity** — reduce latency to users
- **Availability** — not all services available in all regions
- **Pricing** — varies by region

### Availability Zones

One or more data centers close together. Each AZ is **isolated from disaster** (separate power, networking, connectivity).

### Edge Locations (Points of Presence)

Endpoints for AWS used for **caching content**, primarily for [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]] (Amazon's CDN). There are many more edge locations than regions.

## AWS Support Plans

Four levels: **Basic**, **Developer**, **Business**, and **Enterprise**.

### Trusted Advisor

Provides recommendations across 5 categories:
- Cost optimization
- Performance
- Security
- Fault tolerance
- Service limits

**Basic & Developer** core checks: S3 bucket permissions, SG unrestricted ports, IAM use, MFA on root, EBS/RDS public snapshots, service limits.

**Business & Enterprise** get full checks + CloudWatch alarms + API access.

### What is the difference between a Region and an Availability Zone? #card #aws #fundamentals
A Region is a geographic area (e.g., us-east-1) containing 3-6 AZs. An AZ is one or more discrete data centers with redundant power/networking, physically separated within a Region for fault isolation.
