# AWS EC2 - Advanced

Networking, placement groups, hibernate, and advanced [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] features.

## Networking — IP Addresses

### Public vs Private IP

- **Public IP** — identifies device on the internet; changes when instance stops/starts
- **Private IP** — identifies device on internal network; stays constant

### IPv4 vs IPv6

- **IPv4** (32-bit): most common, e.g., `1.160.10.240`
- **IPv6** (128-bit): newer, solves IoT scaling

### Elastic IPs

- Fixed public IPv4 you own (persists through stop/start)
- Attached to one instance at a time; limit **5 per account**
- No charge if attached to a running EC2 with no other Elastic IP
- **Avoid using them** — reflects poor architecture. Use DNS or [[50-59-aws-saa/51.05-aws-elb|Load Balancer]] instead.

## Elastic Network Interface (ENI)

Virtual network card, **bound to a specific AZ**. Automatically attached to EC2s.

Attributes: primary private IPv4, secondary IPv4s, one Elastic IP per private IPv4, one public IPv4/IPv6, SGs, MAC address, source/dest check flag.

Attach types: **hot** (running), **warm** (stopped), **cold** (launching). ENIs are free.

Use cases: management networks, dual-homed instances, low-budget HA.

## Enhanced Networking

High-performance networking using **SR-IOV** (Single Root I/O Virtualization). No additional charge.

- **ENA** (Elastic Network Adapter) — up to **100 Gbps** (preferred)
- **Intel 82599 VF** — up to 10 Gbps (older instances, legacy)

### Elastic Fabric Adapter (EFA)

Accelerates HPC and ML. Enables **OS-bypass** for direct device communication. **Linux only**.

Summary: ENI (basic) → ENA (better) → EFA (HPC/ML + OS bypass).

## EC2 Placement Groups

Intentionally placed groups of interdependent instances. Name must be unique per account. Can't merge PGs or move existing instances in (must redeploy from AMI).

### Clustered

- Single AZ, same rack → **lowest latency**, highest network throughput
- Risk: rack failure = all instances fail
- Use for: HPC, big data jobs needing fast completion
- Recommended: homogenous instance types

### Spread

- Each instance on **distinct physical hardware**
- Can span AZs; max **7 instances per AZ**
- Use for: small number of critical instances needing isolation

### Partitioned

- Up to **7 partitions per AZ**, can span multiple AZs
- Each partition on own rack(s); hundreds of EC2s
- Partition failure doesn't affect other partitions
- Use for: HDFS, HBase, Cassandra

## EC2 Hibernate

Saves RAM to [[50-59-aws-saa/51.04-aws-ec2-storage|EBS]] root volume → **much faster boot** (OS not restarted).

Requirements:

- Root volume must be EBS (not Instance Store), **encrypted**, with enough space for RAM
- Instance RAM < **150 GB**
- Max hibernate: **60 days**
- On-Demand and Reserved only
- Not supported in [[50-59-aws-saa/51.06-aws-asg|ASG]] or ECS

Retains: Instance ID, private IPv4, IPv6, Elastic IPs. Does NOT retain public IP.

## EC2 Nitro

Next-gen platform: better networking (enhanced, HPC, IPv6), higher EBS speed (**64,000 IOPS** vs 32,000 non-Nitro), better security.

## Optimizing CPU

Change core count and threads per core at launch:

- **Decrease cores** — reduce licensing costs when you need high RAM
- **Disable multithreading** — 1 thread/core for HPC

## Capacity Reservations

Reserve EC2 capacity in a specific AZ. No 1/3 year commitment needed. Billed immediately.

## Flashcards

What is the max Elastic IP limit per AWS account? :: 5 per account

EC2 Hibernate saves ==RAM== to the encrypted EBS root volume for much faster boot — max ==60 days== hibernate, ==150 GB== RAM limit.

Placement Group — Clustered ::: All instances in single AZ, same rack — lowest latency, highest throughput, but single point of failure if rack fails

What are the three EC2 networking tiers in order? :: ENI (basic) → ENA (up to 100 Gbps) → EFA (HPC/ML with OS bypass, Linux only)

### When would you use each placement group strategy? #card #aws #ec2-advanced

Cluster: low-latency HPC (same rack, same AZ). Spread: max HA for critical instances (different hardware, max 7/AZ). Partition: large distributed systems like HDFS/Kafka (separate failure domains).

When an EC2 instance is stopped and started, its ==public IP can change==, but its private IP stays constant.

Elastic IPs should be ==avoided== — they reflect poor architectural decisions. Better to use a DNS name or a Load Balancer.

An Elastic IP does not incur charges if it's attached to a ==running EC2 instance== that doesn't have another Elastic IP and is attached to a network interface.

Elastic Network Interface (ENI) :: A virtual network card bound to a specific AZ — provides private IPv4, optional secondary IPs, Elastic IP, public IP, Security Groups, and MAC address

ENI attach types: ==hot== (running), ==warm== (stopped), ==cold== (launching). ENIs themselves are free.

SR-IOV (Single Root I/O Virtualization) :: The technology behind Enhanced Networking — provides higher I/O, lower CPU utilization, higher bandwidth, higher PPS, and lower inter-instance latency at no extra charge

Elastic Fabric Adapter (EFA) :: Network device that accelerates HPC and ML by enabling OS-bypass for direct device communication — only supported on Linux

Spread placement groups can span across AZs but are limited to ==7 instances per AZ==.

Partitioned placement groups support up to ==7 partitions per AZ== and can run ==hundreds of EC2 instances==.

Can you move an existing instance into a placement group? :: No — you can't merge placement groups or move instances. You must redeploy from an AMI into the placement group.

EC2 Hibernate retains: Instance ID, private IPv4, IPv6, and ==Elastic IPs==. It does NOT retain ==public IP== addresses.

EC2 Nitro provides max EBS IOPS of ==64,000==, compared to ==32,000== on non-Nitro instances.

### What data does EC2 Hibernate preserve vs. lose? #card #aws #ec2-advanced

Preserves: Instance ID, private IPv4, IPv6, Elastic IPs, RAM state (saved to encrypted EBS root). Loses: public IP address. Requirements: EBS root must be encrypted with space for RAM, instance RAM < 150GB, max 60 days, On-Demand/Reserved only.

EC2 Capacity Reservations :: Reserve EC2 capacity in a specific AZ with no 1/3 year commitment — get billed immediately upon reservation, capacity access is immediate

### When would you disable multithreading on an EC2 instance? #card #aws #ec2-advanced

For High Performance Computing (HPC) workloads that benefit from 1 thread per CPU core. You can also decrease core count to reduce licensing costs when you need high RAM but few cores.
