# AWS EC2 - Pricing Models

Different [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] pricing models for different workloads.

## Hotel Analogy

- **On-Demand** — walk in, pay full price
- **Reserved** — plan a long stay, get a discount
- **Spot** — bid on empty rooms, highest bidder keeps them, can be kicked out
- **Dedicated Hosts** — book the entire building

## On-Demand

- Fixed rate per hour/second, no commitment
- **Highest cost** but no upfront payment
- Use for: dev/test, short-term, spiky, unpredictable workloads that can't be interrupted

## Reserved Instances

Significant discount (up to 75% off) with 1 or 3 year commitment.

- **Standard Reserved** — up to 75% off
- **Convertible Reserved** — can convert instance types, up to 54% off
- **Scheduled Reserved** — specific time windows
- **Reserved Instance Marketplace** — sell/buy unused Standard RIs
- Best for: steady state or predictable usage

## Spot Instances

Bid for excess capacity — up to **90% discount** (most cost-efficient).

- If spot price > your max price → 2 minutes to decide (stop or terminate)
- If **you** terminate, you pay for partial hour; if AWS terminates, you don't
- **Not suitable for**: databases, critical jobs, persistent workloads
- **Good for**: big data, CI/CD, containerized workloads, HPC, testing, web services

### Spot Termination

- **One-time request** — instance stops/terminates when price exceeds max
- **Persistent request** — instance relaunched when price drops back down
- To fully stop: cancel spot request first, THEN terminate instances

### Spot Block (Deprecated)

1-6 hour continuous run, 30-45% cheaper. No longer available since July 2021.

### Spot Fleets

Set of Spot Instances + (optional) On-Demand Instances targeting a capacity.

Launch pools based on instance type, OS, AZ. Strategies:
- **capacityOptimized** — optimal capacity for launching instances
- **lowestPrice** (default) — cost optimization, short workloads
- **diversified** — distributed across pools (availability, long workloads)
- **InstancePoolsToUseCount** — with lowestPrice, distribute across N pools

## Dedicated Hosts

Physical EC2 server dedicated for your use. Allocated up to 3 years (**most expensive**).

- Use existing server-bound software licenses
- Address compliance requirements
- On-demand or reserved pricing

## Dedicated Instances

Hardware dedicated to one customer, but multiple instances from same account can share it. **Cheaper** than Dedicated Hosts but can't use server-bound licenses.

## Flashcards

Which EC2 pricing model offers up to 90% discount but can be interrupted? :: Spot Instances

Spot Instances are ==not suitable== for databases, critical jobs, or persistent workloads.

Dedicated Hosts ::: Physical EC2 servers dedicated for your use, allocated up to 3 years — the most expensive option, used for server-bound software licenses and compliance

What are the four Spot Fleet strategies? :: capacityOptimized, lowestPrice (default), diversified, and InstancePoolsToUseCount

### How do you choose between Reserved, Savings Plans, and Spot? #card #aws #pricing
Reserved: steady-state workloads (1-3yr commitment). Savings Plans: flexible across instance families (hourly commitment). Spot: fault-tolerant batch/stateless workloads (up to 90% discount, can be interrupted).

On-Demand EC2 :: Fixed rate per hour/second with no commitment — highest cost, no upfront payment, suitable for dev/test and unpredictable workloads

Standard Reserved Instances offer up to ==75%== off On-Demand, while Convertible Reserved Instances offer up to ==54%== off.

Reserved Instance commitments are ==1 or 3== year terms.

Reserved Instance Marketplace :: A platform for buying and selling unused Standard Reserved Instances from third parties or other AWS customers

If a Spot Instance is terminated by AWS due to price exceeding your max, you are ==not charged== for the partial hour. If you terminate it yourself, you ==are charged==.

### What are the steps to terminate a persistent Spot request? #card #aws #pricing
1) First cancel the spot request (to prevent new instances from launching). 2) Then terminate the associated spot EC2 instances. If you only terminate instances without canceling the request, new ones will keep launching.

Spot Fleet :: A set of Spot Instances + optional On-Demand Instances that automatically request instances with the lowest price to meet target capacity

The ==diversified== Spot Fleet strategy distributes across pools for great availability on long workloads, while ==lowestPrice== is the default for cost optimization on short workloads.

Dedicated Instances ::: Instances running on hardware dedicated to one customer — multiple instances from the same account can share hardware, cheaper than Dedicated Hosts but can't use server-bound licenses

### What are the good use cases for Spot Instances? #card #aws #pricing
Big data, data analysis, containerized workloads, CI/CD, web services, HPC, and testing. The key question is "Can your app be terminated within 1-2 minutes notice?"

A Spot Block was used to launch Spot instances for ==1 to 6 hours== continuously at 30-45% less than On-Demand, but is no longer available since July 2021.

### What is the difference between Dedicated Hosts and Dedicated Instances? #card #aws #pricing
Dedicated Hosts: physical server dedicated to you, can use server-bound software licenses, most expensive. Dedicated Instances: hardware dedicated to your account, multiple instances can share it, cheaper, but NO server-bound license support.
