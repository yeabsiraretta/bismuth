# AWS EC2 - Fundamentals

**Elastic Compute Cloud** — resizable compute capacity in the cloud. A [[50-59-aws-saa/51.00-aws-saa-overview#Regional Services|regional service]].

## EC2 Capabilities

- Renting virtual machines ([[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]])
- Storing data on virtual drives ([[50-59-aws-saa/51.04-aws-ec2-storage|EBS]])
- Distributing load across machines ([[50-59-aws-saa/51.05-aws-elb|ELB]])
- Scaling services ([[50-59-aws-saa/51.06-aws-asg|ASG]])

## Creating an EC2 Instance

1. Choose an **AMI** (Amazon Machine Image) — software config template
2. Choose an **Instance Type** (CPU, memory, network)
3. Configure Instance Details (user data scripts, subnet for AZ selection)
4. Add Storage, Add Tags
5. Configure Security Groups, Review and Launch

Subnets are randomized for each AWS account. Termination protection is off by default.

## Bootstrap Script (User Data)

Bash script run at instance startup as **root user**. Used to automate:
- Installing updates and software
- Downloading common files

## Instance Metadata

Available at `169.254.169.254` (link-local address, only accessible from within the instance):
- `curl http://169.254.169.254/latest/user-data` — bootstrap script
- `curl http://169.254.169.254/latest/meta-data/:typeOfData` — instance info
- Can be disabled to prevent misuse

## Security Groups

Firewall controlling traffic in/out of [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] instances.

- **Only contain allow rules** (no explicit deny rules)
- Default: all inbound **blocked**, all outbound **authorized**
- Any number of EC2s can share a SG; limit **5 SGs per EC2**
- Union of allow rules across all attached SGs
- Locked to a **region/[[50-59-aws-saa/51.16-aws-vpc|VPC]] combo**
- **Timeout** = SG issue; **Connection refused** = application error
- SGs can reference other SGs (great for [[50-59-aws-saa/51.05-aws-elb|load balancers]])
- Tip: Maintain one SG specifically for SSH access

Classic ports: SSH (22), HTTP (80), HTTPS (443), RDP (3389).

## Connecting to EC2

- **SSH**: `ssh -i <key>.pem ec2-user@<public-IP>` (protect .pem with `chmod 0400`)
- **EC2 Instance Connect**: Browser-based, depends on SSH inbound SG rule

## Instance Types

Naming convention: e.g., `m5.2xlarge` (family + generation + size).

| Type | Use Case | Key Trait |
|------|----------|-----------|
| **General Purpose** | Web servers, code repos | Balanced compute/memory/network |
| **Compute Optimized** | Batch processing, ML, gaming, HPC | High-performance processors |
| **Memory Optimized** | In-memory DBs, real-time processing | Large data sets in memory |
| **Storage Optimized** | OLTP, data warehousing, NoSQL | High sequential read/write |

## IAM with EC2

- EC2 comes with AWS CLI installed but limited by default
- **Never** `aws configure` on an instance with personal credentials
- Instead, assign an [[50-59-aws-saa/51.01-aws-iam|IAM role]]: Instance → Actions → Security → Modify IAM Role

## Flashcards

What does the EC2 bootstrap script (User Data) run as? :: The root user, executed once at instance startup

Security Groups only contain ==allow== rules — there are no explicit deny rules.

If you get a timeout connecting to EC2, it's a ==Security Group== issue. If you get "connection refused," it's an ==application== error.

EC2 Instance Metadata ::: Data available at `169.254.169.254` from within the instance, providing info like user-data script and instance configuration

### When would you choose each EC2 instance family? #card #aws #ec2
General Purpose (t/m) for web servers, Compute Optimized (c) for batch/ML, Memory Optimized (r/x) for in-memory DBs, Storage Optimized (i/d) for data warehousing, Accelerated (p/g) for GPU workloads.

By default all ==inbound== traffic is blocked and all ==outbound== traffic is authorized in Security Groups.

The limit is ==5== Security Groups per EC2 instance.

Security Groups are locked to a ==region/VPC== combo — you need a new one for each pair.

Security Groups can reference ==other Security Groups== instead of just IP addresses, which is great for load balancers.

EC2 instance metadata IP address :: `169.254.169.254` — a link-local address only accessible from within the instance

What classic TCP ports should you know for the exam? :: SSH (22), HTTP (80), HTTPS (443), RDP (3389)

EC2 termination protection is ==off== by default.

To protect a `.pem` file for SSH :: Run `chmod 0400 <file>.pem` to make it read-only for the owner

EC2 Compute Optimized instance use cases ::: Batch processing, media transcoding, gaming servers, scientific modeling, ML, and HPC

EC2 Memory Optimized instance use cases ::: Relational/non-relational databases, distributed web scale cache stores, in-memory databases, and real-time data processing

EC2 Storage Optimized instance use cases ::: High frequency OLTP, relational and NoSQL databases, in-memory database caches, and data warehousing

### Why should you never enter personal credentials on an EC2 instance? #card #aws #ec2
Anyone with access to that instance could use your credentials. Instead, assign an IAM role to the instance via Actions → Security → Modify IAM Role. Roles are more secure and easier to manage.

By selecting a ==subnet==, you choose which AZ an EC2 instance should be in for that region. Subnets are randomized for each AWS account.
