# AWS Extra Services & ML

Additional services commonly tested on the [[50-59-aws-saa/51.00-aws-saa-overview|SAA]] exam.

## CloudFormation

Infrastructure as Code (IaC). Declarative YAML/JSON templates.

- **Supports almost all AWS resources** — creates infrastructure in the right order
- Cost estimation, resource tagging, savings strategy (dev: auto-delete at 5pm, recreate at 8am)
- Productivity: destroy/recreate infrastructure on the fly
- **Stack Designer** — visual diagram of resources + relationships
- Don't reinvent the wheel — leverage existing templates

## Elastic Beanstalk

PaaS — deploy applications without managing infrastructure.

- Managed service: handles [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]], [[50-59-aws-saa/51.06-aws-asg|ASG]], [[50-59-aws-saa/51.05-aws-elb|ELB]], [[50-59-aws-saa/51.09-aws-databases|RDS]]
- Developer only manages code
- Health monitoring, notifications, metrics
- Supports: Docker, Java, Node.js, Python, Ruby, Go, .NET

## Systems Manager (SSM)

Manage [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] and on-prem infrastructure at scale.

- **Run Command** — execute scripts/commands without SSH (uses SSM Agent)
- **Patch Manager** — automate OS patching
- **Maintenance Windows** — schedule actions
- **Session Manager** — secure shell without SSH port 22 / bastion hosts. Logs to [[50-59-aws-saa/51.19-aws-monitoring|CloudWatch]]/S3.
- **Parameter Store** — see [[50-59-aws-saa/51.15-aws-security#SSM Parameter Store|Security note]]

## AWS Machine Learning

### SageMaker

Fully managed ML service. Build, train, deploy ML models.

### Rekognition

Image/video recognition — facial analysis, celebrity recognition, content moderation, text in images.

### Transcribe

Speech-to-text. Supports automatic language identification, PII redaction.

### Polly

Text-to-speech. Deep learning. **Lexicon** for custom pronunciation. **SSML** for emphasis/breathing.

### Translate

Natural language translation.

### Lex

Same tech as Alexa. Natural language understanding. Build chatbots, call center bots.

### Connect

Cloud contact center. Receive calls, create contact flows. Integrates with Lex + Lambda.

### Comprehend

Natural Language Processing (NLP). Sentiment, key phrases, language detection. **Comprehend Medical** for clinical text.

### Textract

Extract text, handwriting, and data from scanned documents. Forms, tables, PDFs.

### Forecast

ML-based time-series forecasting. 50% more accurate than manual analysis. Retail, financial, resource planning.

### Kendra

ML-powered document search. Natural language queries. Learn from user interactions.

### Personalize

Real-time personalized recommendations. Same tech as Amazon.com. Integrates with websites, apps, SMS.

## Other Key Services

### Storage Gateway

Bridge between on-prem data and [[50-59-aws-saa/51.08-aws-s3|S3]].
- **S3 File Gateway** — NFS/SMB → S3 (Standard, IA, One Zone IA, Intelligent-Tiering)
- **FSx File Gateway** — local cache for frequently accessed data backed by FSx for Windows
- **Volume Gateway** — iSCSI block storage backed by [[50-59-aws-saa/51.08-aws-s3|S3]] snapshots. Cached or stored volumes.
- **Tape Gateway** — virtual tape library backed by S3 + Glacier

### Amazon FSx

Managed third-party file systems:
- **FSx for Windows File Server** — SMB, Windows NTFS, AD integration, multi-AZ
- **FSx for Lustre** — high-performance Linux computing (ML, HPC, video). Seamless S3 integration.
- **FSx for NetApp ONTAP** — multi-protocol (NFS, SMB, iSCSI), point-in-time cloning
- **FSx for OpenZFS** — managed ZFS, NFS compatible, snapshots, compression

### Amazon WorkSpaces

Managed Desktop as a Service (DaaS). Provision Windows/Linux virtual desktops.

### Amazon AppStream 2.0

Application streaming. Stream desktop apps to web browser (no infrastructure needed).

### AWS IoT Core

Connect IoT devices to AWS. Serverless, secure. Integrates with Lambda, [[50-59-aws-saa/51.08-aws-s3|S3]], SageMaker.

## Flashcards

What is the difference between CloudFormation and Elastic Beanstalk? :: CloudFormation is IaC for any AWS resource (you define the infrastructure); Beanstalk is PaaS that auto-manages infrastructure for web apps (you just deploy code)

SSM Session Manager provides secure ==shell access== to EC2 without SSH keys, bastion hosts, or open port 22.

Amazon SageMaker ::: Fully managed service for building, training, and deploying machine learning models at scale

Storage Gateway types? :: S3 File Gateway (NFS/SMB to S3), FSx File Gateway (SMB to FSx), Volume Gateway (iSCSI block, cached or stored), Tape Gateway (virtual tape library to S3/Glacier)

### CloudFormation vs CDK vs Elastic Beanstalk — when to use each? #card #aws #infrastructure
CloudFormation: declarative IaC via JSON/YAML templates (full control). CDK: write infrastructure in TypeScript/Python, synthesizes to CloudFormation (developer-friendly). Beanstalk: PaaS — just deploy code, auto-manages infra (fastest for web apps).

CloudFormation StackSets :: Deploy CloudFormation stacks across multiple AWS accounts and regions with a single operation — used with AWS Organizations

Elastic Beanstalk deployment policies :: All at once (fastest, downtime), Rolling (update in batches), Rolling with additional batch (no reduced capacity), Immutable (new ASG, safest), Blue/Green (full new environment with DNS swap)

SSM Patch Manager :: Automate OS and software patching for EC2 instances — define patch baselines, maintenance windows, and compliance reporting

SSM Run Command :: Execute commands on multiple EC2 instances without SSH — uses IAM for access control, outputs to S3 or CloudWatch

Amazon Rekognition :: ML-based image and video analysis — face detection, object detection, text in images, content moderation

Amazon Transcribe :: Automatic speech-to-text conversion — supports multiple languages, custom vocabularies, and PII redaction

Amazon Polly :: Text-to-speech service — converts text to lifelike speech using deep learning, supports SSML for pronunciation control

Amazon Translate :: Neural machine translation service for text between languages

Amazon Lex :: Build conversational chatbots with natural language understanding — same tech that powers Alexa

Amazon Comprehend :: NLP service for text analysis — sentiment analysis, key phrases, language detection, and topic modeling

Amazon Forecast :: ML-based time-series forecasting — 50% more accurate than manual analysis, used for demand planning

Amazon Kendra :: ML-powered document search with natural language queries — learns from user interactions to improve results

Amazon Personalize :: Real-time personalized recommendations using same tech as Amazon.com

### FSx for Windows vs FSx for Lustre — when to use each? #card #aws #storage
FSx for Windows: SMB protocol, Windows NTFS, Active Directory integration, Multi-AZ — for Windows workloads. FSx for Lustre: high-performance Linux computing (ML, HPC, video), seamless S3 integration — for high-throughput parallel file systems.

WorkSpaces :: Managed Desktop as a Service (DaaS) for provisioning Windows or Linux virtual desktops — pay monthly or hourly

AppStream 2.0 :: Stream desktop applications to any web browser without managing infrastructure — unlike WorkSpaces, streams individual apps not full desktops
