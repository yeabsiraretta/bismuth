# AWS VPC - Advanced

Advanced networking: VPN, Direct Connect, Transit Gateway, and hybrid architectures. Builds on [[50-59-aws-saa/51.16-aws-vpc|VPC basics]].

## Site-to-Site VPN

Encrypted connection between on-premises and AWS over the **public internet**.

- **Virtual Private Gateway (VGW)** — VPN concentrator on AWS side, attached to VPC
- **Customer Gateway (CGW)** — software or physical device on customer side
- If CGW is behind NAT: use NAT-T, use public IP of NAT device
- Must enable **Route Propagation** in VGW's route table

### CloudHub

Connect multiple sites (multiple VPN connections) through a single VGW. Hub-and-spoke model. Traffic goes over public internet.

## Direct Connect (DX)

**Dedicated private connection** from on-prem to AWS. Goes through a DX partner/location.

- Throughput: 1 Gbps or 10 Gbps (or hosted: 50Mbps to 10Gbps via partners)
- Access **public** (S3) and **private** (EC2) resources on same connection
- Supports IPv4 and IPv6
- **Lead time: 1+ months** to establish
- Data NOT encrypted by default (private but not encrypted). Use VPN over DX for encryption.

### Direct Connect Gateway

Connect DX to multiple [[50-59-aws-saa/51.16-aws-vpc|VPCs]] in different regions (same account).

### DX Resiliency

- **Maximum** — two DX connections at two separate locations
- **High** — two connections at one location

## Transit Gateway

**Star hub** connecting thousands of VPCs, VPNs, and DX connections.

- Regional resource, works cross-region via peering
- Route Tables control which VPC can talk to which
- Works with Direct Connect Gateway, VPN connections
- **Only AWS service supporting IP Multicast**
- Can share cross-account using RAM (Resource Access Manager)

### Transit Gateway + DX Architecture

On-prem → DX → Direct Connect Gateway → Transit Gateway → multiple VPCs.

### Increasing VPN Bandwidth

Site-to-Site VPN limit: ~1.25 Gbps. Solution: use **ECMP** (Equal-Cost Multi-Path) routing with Transit Gateway to aggregate multiple VPN tunnels.

## Traffic Mirroring

Capture and inspect network traffic in [[50-59-aws-saa/51.16-aws-vpc|VPC]] non-intrusively. Source ENI → Target (ENI or NLB). For: threat detection, compliance, troubleshooting.

## IPv6

- All IPv6 addresses are public (no NAT needed for IPv6)
- VPC can operate in **dual-stack mode** (IPv4 + IPv6)
- EC2 gets at least a private IPv4 + optional public IPv4 + IPv6
- If can't launch EC2: not an IPv6 problem — create new IPv4 CIDR in subnet
- Use [[50-59-aws-saa/51.16-aws-vpc#Egress-Only Internet Gateway|Egress-Only IGW]] for IPv6 outbound-only access

## AWS Network Firewall

Managed firewall for entire VPC (Layer 3-7). Powered by [[50-59-aws-saa/51.15-aws-security#WAF|Gateway Load Balancer]].

- Inspect all directions: VPC-to-VPC, internet, DX, Site-to-Site VPN
- Supports thousands of rules, IP/port/protocol/domain, regex pattern matching
- Active flow inspection for intrusion prevention (IPS)
- Send logs to [[50-59-aws-saa/51.08-aws-s3|S3]], [[50-59-aws-saa/51.19-aws-monitoring|CloudWatch]], Kinesis Firehose

## Networking Costs

- Use private IP for intra-AZ communication (free)
- Cross-AZ private IP: $0.01/GB each way
- Cross-region: $0.02/GB
- Use same AZ for maximum savings (at cost of HA)
- [[50-59-aws-saa/51.08-aws-s3|S3]] ingress: free. Egress: paid. Use [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]] to reduce S3 egress.
- NAT Gateway: $0.045/hr + $0.045/GB. Use [[50-59-aws-saa/51.16-aws-vpc#VPC Endpoints (PrivateLink)|VPC Endpoints]] to reduce NAT Gateway cost.

## Flashcards

What is the key difference between Site-to-Site VPN and Direct Connect? :: VPN goes over public internet (encrypted, quick setup); Direct Connect is a dedicated private physical connection (higher bandwidth, lower latency, takes 1+ month to establish)

Transit Gateway ::: Hub-and-spoke network topology that connects thousands of VPCs and on-premises networks through a single gateway — supports IP Multicast, cross-region peering, and ECMP for bandwidth scaling

Direct Connect provides a ==dedicated private== connection from on-premises to AWS with speeds from ==1 Gbps to 100 Gbps==.

To reduce NAT Gateway data transfer costs, use ==VPC Endpoints== to access S3 and DynamoDB privately.

### Site-to-Site VPN vs Direct Connect — when to use each? #card #aws #networking
VPN: quick setup (minutes), encrypted over public internet, lower cost, variable latency. Direct Connect: dedicated private connection, consistent low latency, higher bandwidth (up to 100 Gbps), takes weeks to establish.

Virtual Private Gateway (VGW) :: The VPN concentrator on the AWS side — attached to the VPC, one end of a Site-to-Site VPN connection

Customer Gateway (CGW) :: The software or physical device on the customer's on-premises side of a Site-to-Site VPN

AWS VPN CloudHub :: Connects multiple on-premises sites through a single VGW using multiple VPN connections — hub-and-spoke model over the public internet

Direct Connect data is ==NOT encrypted== by default (it's private but not encrypted). For encryption, run a ==VPN over DX==.

Direct Connect lead time is ==1+ months== to establish — not suitable for urgent connectivity needs.

Direct Connect Gateway :: Connects a single Direct Connect to multiple VPCs across different AWS regions within the same account

### What are the DX resiliency options? #card #aws #networking
Maximum resiliency: two separate DX connections at two separate locations. High resiliency: two connections at one location. Each provides redundancy against connection or location failure.

Transit Gateway is the ONLY AWS service that supports ==IP Multicast==.

Transit Gateway can share cross-account using ==RAM (Resource Access Manager)==.

ECMP (Equal-Cost Multi-Path) with Transit Gateway :: Aggregates multiple VPN tunnels to exceed the single Site-to-Site VPN limit of ~1.25 Gbps — each VPN provides 2 tunnels × 1.25 Gbps

VPC Traffic Mirroring :: Non-intrusively captures and inspects network traffic from ENIs — source to target (ENI or NLB), used for threat detection, compliance, and troubleshooting

All IPv6 addresses are ==public== (no private IPv6), so no NAT is needed for IPv6.

If you can't launch an EC2 in a dual-stack VPC, it's ==NOT an IPv6 problem== — create a new IPv4 CIDR in the subnet.

AWS Network Firewall :: Managed Layer 3-7 firewall for the entire VPC — inspects traffic in all directions (VPC-to-VPC, internet, DX, VPN) with support for thousands of rules and IPS

### What are the key networking cost optimization strategies? #card #aws #networking
1) Use private IPs for intra-AZ (free). 2) Cross-AZ costs $0.01/GB. 3) Use VPC Endpoints to avoid NAT Gateway charges for S3/DynamoDB. 4) Use CloudFront to reduce S3 egress costs. 5) S3 ingress is free. 6) Same AZ for max savings (trade-off with HA).
