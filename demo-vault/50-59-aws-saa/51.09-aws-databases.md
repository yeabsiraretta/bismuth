# AWS Databases

Database services across the [[50-59-aws-saa/51.00-aws-saa-overview|AWS]] ecosystem.

## RDS - Relational Database Service

Managed SQL databases: Postgres, MySQL, MariaDB, Oracle, MS SQL Server.

- Managed: provisioning, OS patching, continuous backups (PITR), monitoring dashboards
- Automated backups + manual DB snapshots
- **Multi-AZ** for disaster recovery (synchronous standby)
- **Read Replicas** for read scalability (async, up to 15)
- Can't SSH into RDS instances
- Storage auto-scaling when running low

### Read Replicas vs Multi-AZ

| Feature      | Read Replicas                       | Multi-AZ          |
| ------------ | ----------------------------------- | ----------------- |
| Purpose      | Scale reads                         | Disaster recovery |
| Replication  | Async                               | Sync              |
| Count        | Up to 15                            | 1 standby         |
| Regions      | Same or cross-region                | Same region       |
| Network cost | Free same region, paid cross-region | N/A               |

### RDS Proxy

Fully managed DB proxy. Pools and shares connections. Reduces failover time by 66%. Enforces [[50-59-aws-saa/51.01-aws-iam|IAM]] auth. Great for [[50-59-aws-saa/51.13-aws-serverless|Lambda]] functions (connection pooling).

## Aurora

AWS proprietary DB (MySQL/Postgres compatible). **5x faster** than MySQL, **3x** than Postgres.

- Storage auto-grows (10GB → 128TB)
- Up to 15 read replicas (faster replication than MySQL)
- **6 copies** of data across 3 AZs (4/6 for writes, 3/6 for reads)
- Self-healing with peer-to-peer replication
- Writer endpoint + Reader endpoint (connection load balancing)

### Aurora Features

- **Aurora Serverless** — auto-scaling, pay per second, good for unpredictable workloads
- **Aurora Global Databases** — 1 primary region + up to 5 secondary (read) regions, <1s replication, promotes secondary in <1 minute
- **Aurora Machine Learning** — integrate with SageMaker/Comprehend via SQL

## ElastiCache

In-memory database (Redis or Memcached). Sub-millisecond latency.

- **Redis** — Multi-AZ with failover, read replicas, data durability (AOF), backup/restore, sets/sorted sets
- **Memcached** — multi-node partitioning (sharding), no HA, no backup, multi-threaded

Use cases: caching, session stores, gaming leaderboards, [[50-59-aws-saa/51.13-aws-serverless|real-time]] analytics.

## DynamoDB

Fully managed, serverless NoSQL. Single-digit ms latency. Auto-scaling.

- **Tables** with primary key (partition key or partition key + sort key)
- Max item size: 400KB
- **Provisioned** (plan capacity) or **On-Demand** (pay per request, auto-scale)
- **DynamoDB Accelerator (DAX)** — in-memory cache, 10x performance, 5 min TTL default
- **DynamoDB Streams** — ordered stream of item modifications (create/update/delete), triggers [[50-59-aws-saa/51.13-aws-serverless|Lambda]]
- **Global Tables** — multi-region, active-active replication (requires Streams enabled)
- TTL — auto-delete items after expiry
- Supports transactions (ACID across multiple tables)

## Redshift

Data warehouse based on Postgres. **OLAP** (analytics/data warehousing, not OLTP).

- 10x better performance than other warehouses, scales to PBs
- Columnar storage with massively parallel query execution
- Pay per instance provisioned
- SQL interface
- Better to do large batch inserts
- vs [[50-59-aws-saa/51.18-aws-data-analytics|Athena]]: faster queries/joins/aggregations thanks to indexes

## Other Database Services

### Neptune

Fully managed **graph database**. Use for: social networking, knowledge graphs, fraud detection. Multi-AZ, clustering.

### Amazon QLDB

**Ledger** database — immutable, cryptographically verifiable transactions. 2-3x better than blockchain frameworks. Centralized (not blockchain).

### Amazon Timestream

**Time-series** database. IoT, trillions of events/day. 1000x faster, 1/10th cost of relational DB. Serverless.

### How do you choose between RDS, Aurora, DynamoDB, and ElastiCache? #card #aws #databases

RDS/Aurora: relational SQL with joins and transactions. DynamoDB: key-value/document NoSQL for high-throughput, single-digit ms latency. ElastiCache: in-memory caching (Redis/Memcached) for session stores and hot data.

RDS supports up to ==5 Read Replicas== within AZ, cross-AZ, or cross-region. Aurora supports up to ==15 Read Replicas==.

RDS Multi-AZ is for ==disaster recovery== (sync replication to standby), not for read scaling — it provides automatic failover.

Aurora is ==5x faster== than MySQL and ==3x faster== than PostgreSQL on RDS.

Aurora storage automatically grows in increments of ==10 GB== up to ==128 TB==.

Aurora Global Databases :: Cross-region read replicas with replication lag under 1 second — can promote a secondary region for disaster recovery with RTO < 1 minute

DynamoDB is a ==serverless== NoSQL database with single-digit millisecond latency that supports key-value and document data models.

DynamoDB capacity modes: ==Provisioned== (plan capacity with RCU/WCU, auto-scaling) and ==On-Demand== (automatic scaling, pay per request, 2.5x more expensive).

DynamoDB Global Tables :: Multi-region, multi-active replication with read/write capability in any region — requires DynamoDB Streams to be enabled

### ElastiCache Redis vs Memcached — when to use each? #card #aws #databases

Redis: Multi-AZ with failover, read replicas, data persistence, backup/restore, Sets and Sorted Sets. Memcached: multi-node sharding (partitioning), no HA, no persistence, multi-threaded — use when you just need simple caching.

ElastiCache can be used to store ==session data== so that stateless applications can redirect users to different instances.

DocumentDB :: AWS implementation of MongoDB (NoSQL document database) — fully managed, HA with replication across 3 AZs, auto-scaling storage up to 64 TB

Neptune :: Fully managed graph database — used for social networks, knowledge graphs, fraud detection, and recommendation engines

Amazon Keyspaces :: Serverless Apache Cassandra-compatible database — managed, scales tables up/down automatically, on-demand or provisioned capacity

Amazon Timestream :: Fully managed serverless time-series database — built-in time-series analytics functions, 1000x faster than relational databases for time-series queries

RDS Proxy :: Fully managed database proxy that pools and shares connections — reduces failover time by 66%, enforces IAM authentication, never publicly accessible (only from VPC)
