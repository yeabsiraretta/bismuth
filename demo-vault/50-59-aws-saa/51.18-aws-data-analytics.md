# AWS Data & Analytics

Analytics and data processing services in [[50-59-aws-saa/51.00-aws-saa-overview|AWS]].

## Athena

Serverless query service for [[50-59-aws-saa/51.08-aws-s3|S3]] data using standard SQL.

- Supports CSV, JSON, ORC, Avro, Parquet
- $5 per TB scanned
- **Performance tip**: use columnar formats (Parquet/ORC), partition data, compress, use larger files (>128MB)
- **Federated queries**: use Lambda connectors to query other sources (RDS, DynamoDB, on-prem)
- Use cases: BI, analytics, [[50-59-aws-saa/51.16-aws-vpc#VPC Flow Logs|VPC Flow Logs]], [[50-59-aws-saa/51.19-aws-monitoring#CloudTrail|CloudTrail]], [[50-59-aws-saa/51.05-aws-elb|ELB]] logs, cost analysis

## Glue

Managed **ETL** (Extract, Transform, Load) service. Serverless.

- **Glue Data Catalog** — central metadata repo scanned by Glue Crawlers
- Other services query Catalog: Athena, [[50-59-aws-saa/51.09-aws-databases#Redshift|Redshift]] Spectrum, [[50-59-aws-saa/51.18-aws-data-analytics#EMR|EMR]]
- **Glue Job Bookmarks** — prevent reprocessing old data
- **Glue Elastic Views** — combine and replicate data across multiple data stores (SQL, uses CDC)
- **Glue DataBrew** — clean and normalize data with pre-built transformations
- **Glue Studio** — visual ETL workflow creation
- **Glue Streaming ETL** — run on streaming data (Kinesis, Kafka, MSK) on Apache Spark

### Convert Data Format

Common pattern: S3 (CSV) → Glue ETL → S3 (Parquet) → Athena (cheaper queries)

## EMR - Elastic MapReduce

Managed Hadoop clusters (hundreds of [[50-59-aws-saa/51.02-aws-ec2-fundamentals|EC2]] instances). Supports Spark, HBase, Presto, Flink.

- Auto-scaling, integrated with [[50-59-aws-saa/51.02-aws-ec2-pricing#Spot Instances|Spot Instances]]
- Use cases: data processing, ML, web indexing, big data

### Node Types

- **Master** — manages cluster, health checks (long-running)
- **Core** — runs tasks, stores data (long-running)
- **Task** — only runs tasks, no storage (optional, usually Spot)

## QuickSight

Serverless BI service for interactive dashboards.

- Auto-scales, per-session pricing
- Integrates with: [[50-59-aws-saa/51.09-aws-databases|RDS]], Aurora, Athena, [[50-59-aws-saa/51.09-aws-databases#Redshift|Redshift]], [[50-59-aws-saa/51.08-aws-s3|S3]], OpenSearch, Timestream, Jira, Salesforce
- **SPICE** — in-memory computation engine for imported data
- **Column-Level Security (CLS)** via Enterprise edition
- Users and groups (Enterprise only) — not [[50-59-aws-saa/51.01-aws-iam|IAM]]

## OpenSearch (Elasticsearch)

Search and analytics engine. Complement to [[50-59-aws-saa/51.09-aws-databases|DynamoDB]] (which can't search by partial/full text).

- Managed or serverless (auto-scaling)
- Does NOT natively support SQL (can use plugin)
- Ingestion from: [[50-59-aws-saa/51.12-aws-messaging|Kinesis Firehose]], IoT, [[50-59-aws-saa/51.19-aws-monitoring|CloudWatch]] Logs
- Security via [[50-59-aws-saa/51.13-aws-serverless#Cognito|Cognito]], [[50-59-aws-saa/51.01-aws-iam|IAM]], [[50-59-aws-saa/51.15-aws-security#KMS|KMS]], TLS
- Dashboards via OpenSearch Dashboards (Kibana)

## Lake Formation

Build a data lake in days. Centralized permissions, deduplication.

- Sources: [[50-59-aws-saa/51.08-aws-s3|S3]], on-prem DBs, [[50-59-aws-saa/51.09-aws-databases|RDS]], Aurora, NoSQL
- Consumers: Athena, Redshift, EMR, Spark
- **Fine-grained access control** (row/column level) — centralized, replaces S3/DB/IAM policies
- Built on Glue under the hood

## Flashcards

What is Amazon Athena? :: Serverless query service that analyzes S3 data using standard SQL — pay per TB scanned, supports CSV, JSON, Parquet, ORC

AWS Glue is a managed ==ETL== (Extract, Transform, Load) service that also maintains the ==Glue Data Catalog== for Athena, Redshift, and EMR.

Amazon Redshift ::: Petabyte-scale data warehouse based on PostgreSQL — columnar storage, MPP, not for OLTP but for OLAP (analytics/data warehousing)

To improve Athena performance and reduce cost, use ==columnar formats== (Parquet/ORC) and ==partition== data in S3.

### Athena vs Redshift vs EMR — when to use each? #card #aws #analytics

Athena: ad-hoc SQL queries on S3 (serverless, pay per scan). Redshift: persistent data warehouse for complex OLAP (provisioned cluster). EMR: big data processing with Hadoop/Spark/Hive (managed clusters for ETL, ML, streaming).

Redshift is ==NOT multi-AZ== — it runs in a single AZ. Use Redshift Spectrum to query S3 data without loading it.

Redshift Enhanced VPC Routing :: Forces all COPY and UNLOAD traffic between Redshift cluster and S3 through the VPC — enables VPC features like flow logs, NACLs, and VPC endpoints

Amazon EMR :: Managed Hadoop clusters (Spark, HBase, Presto, Flink) for big data processing — auto-scales, spot instances for cost savings

Amazon OpenSearch (successor to ElasticSearch) :: Full-text search and analytics engine — commonly used with Kinesis Data Firehose, Lambda, and CloudWatch Logs for log analytics

Glue Data Catalog :: Central metadata repository that stores table definitions and schemas — shared across Athena, Redshift Spectrum, and EMR

AWS Lake Formation :: Makes it easy to set up a secure data lake — built on S3, provides fine-grained access control, combines data from multiple sources

Amazon QuickSight :: Serverless BI (Business Intelligence) service for interactive dashboards and visualizations — integrates with RDS, Aurora, Athena, Redshift, S3

Amazon MSK (Managed Streaming for Apache Kafka) :: Fully managed Apache Kafka service — alternative to Kinesis with Kafka API compatibility, runs in your VPC

Athena Federated Query :: Query data across multiple sources (RDS, DynamoDB, Redshift, CloudWatch, on-prem) using Lambda-based data source connectors — not limited to S3

Kinesis Data Analytics :: Real-time analytics on streaming data using SQL or Apache Flink — takes input from Kinesis Data Streams or Firehose
