# AWS Serverless

Serverless compute and application services.

## Lambda

Run code without provisioning servers. Pay per request + compute time.

- Up to **10GB RAM**, 15 min timeout
- Supports: Node.js, Python, Java, C#, Go, Ruby, custom runtime
- **Event-driven** — invoked by AWS services or direct API calls
- Auto-scaling (up to 1,000 concurrent executions default)
- Free tier: 1M requests + 400,000 GB-seconds/month

### Lambda Integrations

- [[50-59-aws-saa/51.08-aws-s3|S3]] event notifications
- [[50-59-aws-saa/51.12-aws-messaging|SQS/SNS/Kinesis]] processing
- [[50-59-aws-saa/51.19-aws-monitoring|CloudWatch]] Events / [[50-59-aws-saa/51.19-aws-monitoring#EventBridge|EventBridge]]
- API Gateway
- [[50-59-aws-saa/51.09-aws-databases|DynamoDB]] Streams
- [[50-59-aws-saa/51.14-aws-containers|ECS]] tasks

### Lambda Networking

- Default: launched outside your [[50-59-aws-saa/51.16-aws-vpc|VPC]] (access public internet, not VPC resources)
- **Lambda in VPC**: access VPC resources (RDS, ElastiCache). Needs NAT Gateway for internet access.

### Lambda with RDS Proxy

Use [[50-59-aws-saa/51.09-aws-databases#RDS Proxy|RDS Proxy]] for connection pooling — Lambda opens many connections otherwise.

## API Gateway

Fully managed API creation, publishing, and security.

- Supports REST, HTTP, and WebSocket APIs
- **Serverless** — no infrastructure to manage
- Rate limiting, API keys, authentication
- Swagger/OpenAPI import
- Transform and validate requests/responses
- Cache API responses

### Endpoint Types

- **Edge-optimized** (default) — routed through [[50-59-aws-saa/51.11-aws-cloudfront|CloudFront]] edge locations
- **Regional** — for clients in same region
- **Private** — accessible only from [[50-59-aws-saa/51.16-aws-vpc|VPC]] using interface endpoint

### Security

- [[50-59-aws-saa/51.01-aws-iam|IAM]] policies (for internal/cross-account)
- Cognito User Pools (for external users)
- Lambda Authorizer (custom logic)
- HTTPS via [[50-59-aws-saa/51.15-aws-security#AWS Certificate Manager|ACM]]

## Cognito

User identity and access management for web/mobile apps.

### Cognito User Pools (CUP)

- Sign-in for app users (serverless user database)
- Username/email/phone + password
- MFA, email verification, social logins (Facebook, Google, SAML)
- Returns **JWT tokens**
- Integrates with API Gateway and [[50-59-aws-saa/51.05-aws-elb|ALB]]

### Cognito Identity Pools (Federated Identity)

- Provide **temporary AWS credentials** to access AWS resources directly
- Identity source: CUP, 3rd-party, guest
- [[50-59-aws-saa/51.01-aws-iam|IAM]] policies applied to credentials

## Step Functions

Orchestrate serverless workflows as visual state machines.

- Sequence Lambda functions and other AWS services
- Features: branching, parallel execution, error handling, retries, human approval
- Max execution: **1 year**
- Use for: order fulfillment, data processing, ETL, any workflow

## SAM - Serverless Application Model

Framework for developing and deploying serverless apps.

- YAML configuration (extension of [[50-59-aws-saa/51.20-aws-extra-services#CloudFormation|CloudFormation]])
- `sam build`, `sam deploy` for packaging
- Local testing with Lambda-like emulation
- Supports Lambda, API Gateway, DynamoDB tables

## Flashcards

What are the Lambda resource limits? :: 128 MB to 10 GB RAM, up to 15 min execution, 512 MB to 10 GB /tmp disk, 1000 concurrent executions (can increase)

Lambda ==Provisioned Concurrency== eliminates cold starts by pre-initializing execution environments.

API Gateway ::: Fully managed service that creates REST, HTTP, and WebSocket APIs — handles authentication, throttling, caching, and can proxy to Lambda, HTTP backends, or AWS services

Cognito User Pools vs Identity Pools? :: User Pools handle sign-in/sign-up (authentication); Identity Pools provide temporary AWS credentials for direct access to AWS resources (authorization)

### What are Lambda's key limits and how do you work around them? #card #aws #serverless
Memory: 128MB-10GB, timeout: max 15 min, deployment: 50MB zipped/250MB unzipped. For larger packages use Lambda Layers or container images. For long-running tasks, use Step Functions or ECS/Fargate instead.

Lambda default concurrent executions limit is ==1000== per region (can be increased).

Lambda can run inside a ==VPC== to access private resources like RDS and ElastiCache — requires VPC config with subnets and security groups.

Lambda@Edge :: Run Lambda functions at CloudFront edge locations for request/response customization — used for A/B testing, bot filtering, user authentication at the edge

Lambda Layers :: Share common code/libraries across multiple Lambda functions — up to 5 layers per function, helps reduce deployment size

API Gateway throttling default is ==10,000 requests/second== with a burst of ==5,000==.

API Gateway can cache responses for a ==TTL of 300 seconds== (default) to reduce backend calls.

Cognito User Pools return ==JWT tokens== after authentication and integrate with API Gateway and ALB.

Step Functions :: Orchestrate serverless workflows as visual state machines — supports branching, parallel execution, error handling, retries, and human approval with max execution of 1 year

SAM (Serverless Application Model) :: YAML framework extending CloudFormation for serverless apps — `sam build` and `sam deploy` for packaging, supports local Lambda testing

### Lambda invocation models — what are they? #card #aws #serverless
Synchronous: caller waits for response (API Gateway, ALB). Asynchronous: Lambda queues event and retries twice on failure (S3, SNS, EventBridge). Event Source Mapping: Lambda polls from stream/queue (SQS, Kinesis, DynamoDB Streams).
